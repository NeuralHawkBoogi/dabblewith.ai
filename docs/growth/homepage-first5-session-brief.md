## Growth move

Convert one warm responder into a 20-minute builder walkthrough around a concrete workflow sample.

The session should produce one reusable proof point:

- anonymized workflow title,
- before/after workflow map,
- first prompt/process/checklist,
- safety/privacy boundary,
- next test,
- optional referral or group-owner intro.

## Copy-ready slot confirmation

> Perfect — let’s keep this very concrete.
>
> I can do a 20-minute builder walkthrough around your workflow. Please send before the call:
> 1) the input you start with,
> 2) the output you want,
> 3) tool/stack constraints,
> 4) what must stay private,
> 5) one example where the current process breaks.
>
> I’ll use that to show a practical first version, not a generic AI overview.

## Run-of-show

1. 2 min — restate the workflow and safety boundary.
2. 4 min — map current input → decision → output steps.
3. 6 min — build the first prompt/process/checklist live.
4. 4 min — add guardrails: private data, human review, failure cases.
5. 3 min — define the reusable template and next test.
6. 1 min — ask for one referral or one group-owner intro only if the session was useful.

## Recap template

```text
Builder walkthrough recap

Workflow: [anonymized title]
Before: [manual/repetitive step]
After: [AI-assisted first version]
Useful output: [prompt / checklist / workflow map]
Safety boundary: [what stayed private + human review point]
Next test: [one small validation step]
Reusable for dabblewith.ai: [mini-session theme / blog note / next DM hook]
Referral ask: [one person/group that has a similar workflow]
```

## Decision rules

- Session completed + useful output → send next 3–5 warm DMs using the anonymized before/after as the hook.
- Strong workflow but no slot → send one nudge, then park.
- Group-owner/admin pain appears → route to `/community-bot/` readiness/design-partner flow.
- No workflow sample after 24h → rewrite the invite around one narrower promise instead of broadening outreach.

## Privacy guardrails

- Do not publish raw WhatsApp text, message IDs, full phone numbers, or private workflow data.
- Keep recap anonymized unless the responder explicitly approves attribution.
- Ask the community-bot readiness question only when a real group-owner/admin context appears.
