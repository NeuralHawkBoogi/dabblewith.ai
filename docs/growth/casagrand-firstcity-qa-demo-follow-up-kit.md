# Casagrand First City — QA Demo Follow-up Kit

Date: 2026-05-25
Public page: `https://dabblewith.ai/casagrand-firstcity/qa-demo-follow-up/`

## Why this exists

The live Casagrand report currently shows one real resident signal after owner/test exclusion: QA/coding/student-project/event interest. The right next growth move is not a broad IT-group post yet. It is to convert that first QA signal into concrete proof, referrals, and possibly a community-bot validation lead.

This kit starts after either:

- the resident shares an anonymized QA/workflow sample, or
- the resident agrees to a 20-minute QA workflow walkthrough.

## Operator sequence

1. Confirm a 20-minute walkthrough and repeat the privacy guardrail: no company names, customer data, credentials, screenshots, or confidential data.
2. Run the mini-demo: feature/bug note → test cases → assumptions → edge cases → regression checklist → human review points.
3. Send the recap immediately while the usefulness is fresh.
4. Ask for 1–2 QA/dev/student referrals and one group-owner/admin intro if relevant.
5. Track only last4, route, segment, sample/demo status, topic vote, slot vote, referral count, community-bot probe, and next action.

## Decision thresholds

- Sample received but no slot: send one confirmation nudge.
- Demo completed and useful: ask for two referrals.
- Three QA/dev/student signals: run a topic/date poll.
- One group-owner/admin interest: send the bot-readiness audit.
- Five or more total resident signals: prepare clubhouse slot.
- No follow-through after 24 hours: continue the narrow 5-DM sprint before any broad IT-group post.

## Copy blocks

Use the public page copy buttons for:

- slot confirmation
- post-demo recap
- referral + poll ask
- privacy-safe manual tracker row

## Privacy guardrails

- Redact phone numbers except last four digits.
- Do not store raw WhatsApp messages, raw webhook payloads, screenshots, company data, credentials, or tokens.
- Do not imply this is official Casagrand association software.
- Treat community-bot interest as design-partner validation until a human approves any external commitment.
