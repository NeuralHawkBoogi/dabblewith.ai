## 3. Workflow draft
Problem: You want to share weekly progress publicly but don't know which numbers honestly reflect momentum versus vanity.

Outcome: Each Friday you publish a 6-line update with the same metrics, plus one lesson, in 30 minutes.

Tools today:
- GA or analytics export
- Plaintext notes
- Newsletter or X thread
- Any LLM in summarization-only mode

Steps:
1. Pick 6 metrics: visitors, returning visitors, signups, replies/conversations, workflow submissions or product actions, and MRR (or $0 if true).
2. Add 2 qualitative lines: what experiment ran, what surprised you.
3. Add 1 explicit failure or thing that didn't work.
4. Use an LLM to compress the raw notes into the same 6-line template every week.
5. Publish on the same day and time every week so readers know when to expect it.
