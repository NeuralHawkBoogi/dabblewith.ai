## 3. Workflow draft
Problem: You're running an internal AI workshop and don't know what each attendee actually wants to build, so the session ends up generic.

Outcome: Each attendee shows up with a specific workflow problem, sample inputs, and a publish-or-not decision before the workshop starts.

Tools today:
- WhatsApp
- Email
- Spreadsheet
- Any LLM with a draft-only review pass

Steps:
1. Define the workshop's single outcome — a published workflow, a working draft, or a reviewed prompt.
2. Send an intake message asking: role, one workflow they want, sample input they can share safely, what they'd never automate.
3. Use an LLM in draft-only mode to cluster responses into 3-5 themes. Read every response yourself.
4. Pre-write a starter workflow for each cluster so no one starts at a blank screen.
5. Confirm with each attendee 24 hours before the session that the starter fits their problem.

## 4. Human review gate
An organizer must read every intake reply before clustering. LLM may suggest themes but cannot decide who attends or what is exposed.
