Recovery tracker wizard · updated 2026-05-28

Fill the recovery batch tracker without leaking anything.

After sending the stale responder nudge and the five warm narrow-discovery DMs, use this wizard to write one combined private tracker — last4 only, no raw phone numbers, names, message bodies, screenshots, or tokens — and rerun the privacy-safe campaign report.

Back to send sheet Open checklist
step 1

Send first

Use the recovery send sheet to send the one-time nudge and the five warm DMs. Do not start the tracker until messages are out.

step 2

Write the template

Generate the combined recovery-batch template under private/. Every row starts as no_reply so positive outcomes are only counted from real replies.

step 3

Fill, then rerun

Edit only the allowed fields per row, save, and rerun the campaign report. The report renders the Recovery batch follow-up section with the route-specific next page.

1) Generate the template

Creates one combined private tracker for the whole recovery batch (stale nudge, warm-intro ask, two QA/dev DMs, two Excel/workflow DMs, one group-owner/admin DM).

mkdir -p private node scripts/casagrand-campaign-report.js --write-recovery-batch-template private/casagrand-recovery-batch.json

2) Fill one row per DM

Each tracker row keeps last4 only. No raw phone numbers, names, WhatsApp text, screenshots, files, message IDs, or tokens.

{ "last4": "0000", "segment": "stale_responder | warm_intro | qa_dev_student | excel_workflow | group_owner", "route": "stale_responder_recovery_batch", "problemType": "qa_test_plan | coding_helper | excel_report | office_workflow | community_bot | none", "followUpSent": true, "outcome": "no_reply | replied_topic | replied_sample | replied_referral | replied_group_owner | declined", "nextAction": "qa_walkthrough | excel_walkthrough | referral_sprint | bot_readiness | date_lock | continue_narrow_discovery", "note": "short sanitized note — no raw WhatsApp text, no names, no numbers beyond last4" }

3) Rerun the privacy-safe report

The rerun renders the dedicated Recovery batch follow-up section and recommends the next page.

node scripts/casagrand-campaign-report.js --runtime-dir /home/clawdbot/dabblewith-whatsapp/data --date 2026-05-28 --exclude-last4 2585 --manual-tracker private/casagrand-recovery-batch.json

Never put in the tracker

  • Full phone numbers (only the last 4 digits).
  • Resident or admin names.
  • Raw WhatsApp message bodies, screenshots, or files.
  • Tokens, message IDs, webhook payloads, or credentials.
  • Spreadsheets, resumes, work data, or company info.
