Casagrand manual tracker console · 2026-05-25

Turn private DM replies into a clean 24-hour growth decision.

Use this after the five narrow Casagrand DMs are sent. It keeps private WhatsApp replies out of public docs, captures only last-four-safe outcomes, and makes the next report rerun deterministic.

why now

Bot logs may miss private DMs

The five-DM sprint can happen inside Boogi’s personal WhatsApp context. This console preserves evidence without requiring raw chats in the runtime bot logs.

privacy rule

Last four only

Store segment, route, one short problem phrase, follow-up state, and next action. Never store full phone numbers, raw messages, names, tokens, or message IDs.

decision gate

24-hour rerun

After the sprint window closes, rerun the report with the manual tracker attached and use the thresholds instead of gut feel.

Operator flow

1

Generate the starter file before sending DMs so the five rows are pre-shaped and private-safe.

2

Fill sprintStartedAt and reportRerunDueAt once the first DM is sent.

3

Update only route, problem, follow-up, and next action as replies arrive. Keep problem phrases under eight words.

4

Attach the tracker to the campaign report after 24 hours. Manual evidence should override no-signal bot logs only when it contains concrete replies.

5

Commit only sanitized docs/reports. Do not commit the filled private tracker unless Boogi explicitly wants a redacted artifact stored.

Decision thresholds

0 concrete repliesrewrite hook before broad post
1–2 concrete problemssend 5 more narrow DMs
3+ topic votes/referralsopen date/topic poll
2+ admin painsbook bot-readiness calls
10+ interested residentslock clubhouse intro

If evidence is mixed, prefer another five DMs over a 600-member broad post. The campaign is validating demand, not maximizing impressions.
