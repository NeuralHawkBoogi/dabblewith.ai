Anonymized example input

Feature note, anonymized: When a resident submits an event RSVP form, the system should validate required fields, prevent duplicate RSVPs for the same phone/email, show a confirmation screen, and let the organizer export the attendee list before the event. Constraints: - no real names or phone numbers in this sample - use fake fields only - mark product questions clearly

AI-by-Doing output sample

Test plan draft: Assumptions to confirm: 1. What counts as duplicate: phone, email, or both? 2. Can a resident edit an RSVP after submission? 3. Who can export the attendee list? Happy path: - submit valid RSVP and see confirmation - organizer export includes expected fields - second distinct resident can RSVP successfully Negative / edge cases: - missing required name/slot/contact field - duplicate phone/email attempt - malformed email or short phone-like value - export attempted by non-organizer - slow network during confirmation Regression checklist: - RSVP source tag preserved - analytics event fires once - no full phone number shown in public view - export excludes private notes by default
