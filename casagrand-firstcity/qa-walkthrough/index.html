Casagrand QA walkthrough · 2026-05-26

Run the first QA workflow walkthrough like a conversion experiment.

This 20-minute host script closes the gap between the proof asset and the referral sprint: confirm the anonymized task, build one reusable QA workflow live, ask for one referral, and log only privacy-safe evidence.

do first

Use anonymized input

Confirm the resident removed employer names, customers, ticket IDs, screenshots, logs, credentials, and real phone/email values before the walkthrough starts.

conversion goal

One reusable workflow

The goal is not a polished product demo. It is one useful QA prompt/checklist the resident would forward to another QA/dev/student neighbor.

next gate

Referral or topic vote

End with exactly one referral ask and one topic/slot vote. Three total resident signals unlock the date-lock kit.

20-minute run of show

0

Privacy check · 2 min

Repeat the no-confidential-data rule and use a synthetic fallback if anything looks sensitive.

1

Task framing · 3 min

Ask what the QA task is, when it repeats, and what “good enough to reuse” means.

2

Build live · 8 min

Convert the note into assumptions, happy paths, edge cases, negative tests, and a regression checklist.

3

Human review · 4 min

Ask the resident to mark what is useful, wrong, missing, or risky for real QA work.

4

Referral close · 3 min

Ask for one QA/dev/student referral and one preferred slot/topic before ending.

Opening script

Thanks for doing this. Goal for the next 20 minutes: take one anonymized QA/coding task and turn it into a reusable AI-assisted checklist. Before we start: please do not share employer names, customer data, screenshots, credentials, production logs, ticket IDs, or full phone/email values. If the example is sensitive, we will switch to a synthetic version. At the end I will ask two quick things: whether this is useful enough for a Casagrand mini-session, and one resident referral if yes.
