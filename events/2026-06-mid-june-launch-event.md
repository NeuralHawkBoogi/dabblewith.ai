# dabblewith.ai — Mid-June Launch Event Plan

## Working title
**AI by Doing: Build Your First Practical AI Workflow**

## Recommended date
**Saturday, 13 June 2026** or **Sunday, 14 June 2026**

Recommended slot: **10:30 AM – 1:00 PM IST**

Why weekend morning:
- Easier for working professionals.
- Enough energy for hands-on learning.
- Leaves afternoon for networking/follow-ups.

## Event thesis
The first event should prove the dabblewith.ai promise:

> This is not another AI gyaan session. We will build, break, review, and reuse real AI workflows.

The event should show that the community itself is AI-operated and human-guided.

## Target audience
Primary:
- Chennai IT professionals
- Founders/operators
- Product managers
- Engineers curious about AI agents
- People who use ChatGPT but do not yet have repeatable workflows

Initial target size:
- **30–50 people** for quality
- Cap at **75** if venue is easy and there is strong interest

## Format
**In-person, workshop-first, free first event**

Suggested structure: 2.5 hours

### 0:00–0:10 — Welcome
- What is dabblewith.ai?
- AI-operated, human-guided community
- Ground rule: practical output > hype

### 0:10–0:30 — Demo: The community OS
Show:
- Website autopilot
- Session queue
- Blog/content engine
- WhatsApp/member signal plan
- Human approval gates

### 0:30–1:10 — Live build: one practical AI workflow
Build one workflow live, e.g.:
- Meeting transcript → decisions → tasks → follow-up message
- Candidate interview notes → hiring signal → next-step ticket
- Founder idea → landing page outline → validation plan

Recommended first workflow:
**Meeting/voice notes → action tickets → follow-up plan**

Why:
- Everyone understands it.
- Demonstrates AI usefulness immediately.
- Connects to NeoSapien/OpenClaw story naturally.

### 1:10–1:30 — Break + signal collection
Attendees submit:
- Their role
- One workflow they want automated
- One AI tool they currently use
- Whether they want future sessions

### 1:30–2:05 — Breakout: design your own AI workflow
Small groups fill a simple template:
- Input
- Desired output
- AI/tool step
- Human review gate
- Failure mode
- Reuse plan

### 2:05–2:25 — Show 3 examples
Pick 3 attendee workflows and critique/improve live.

### 2:25–2:30 — Close
- Join WhatsApp/community list
- Vote for next event
- Share recap/playbook promise

## Venue options
### Option A — Theater / screening room
Best if available around ₹1,500 for 3 hours.
Pros:
- Memorable first event
- Good projection/audio
- Feels like a launch
Cons:
- Less convenient for tables/hands-on work

### Option B — Office / coworking classroom
Pros:
- Better for workshop
- Easier logistics
- Lower cost
Cons:
- Less “event” feel

Recommendation:
Use **office/coworking** if the goal is hands-on quality. Use **theater** only if we can still handle laptop/workshop interaction.
